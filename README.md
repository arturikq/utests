# План тестирования модуля `rectangle`

## 1. Цели и задачи тестирования
- Проверка корректности вычисления площади (`area`) и периметра (`perimeter`) прямоугольника.  
- Убедиться, что функции корректно обрабатывают:
  - нулевые значения сторон,
  - квадратные прямоугольники,
  - обычные прямоугольники с различными сторонами.  
- Обеспечить стабильность функций при последующих изменениях кода.  

## 2. Описание тестируемого продукта
- **Продукт:** модуль `rectangle`, предоставляющий функции для работы с прямоугольниками.  
- **Функции:**
  - `area(a, b)` — вычисляет площадь прямоугольника с сторонами `a` и `b`.
  - `perimeter(a, b)` — вычисляет периметр прямоугольника с сторонами `a` и `b`.
- **Требования:**
  - Функции возвращают численные значения.
  - Корректно работают для положительных и нулевых значений сторон.  

## 3. Область тестирования
- Функция `area(a, b)` с разными комбинациями сторон: 0, равные, разные.
- Функция `perimeter(a, b)` с разными комбинациями сторон: 0, равные, разные.
- Проверка, что функции возвращают ожидаемые значения.  

## 4. Стратегия тестирования
- Используется модульное тестирование с использованием `unittest`.
- Тестовые кейсы проверяют:
  - граничные значения (0),
  - обычные значения (разные стороны),
  - квадраты (равные стороны).
- Методы тестирования:
  - Функциональное тестирование (`assertEqual`).
  - Тестирование устойчивости функций к нулевым значениям.
- Запуск тестов через `main.py` для автоматизации.  

## 5. Критерии приемки
- Все тесты проходят успешно без ошибок.
- Функции возвращают корректные значения для всех проверяемых случаев.
- История репозитория содержит коммиты с добавлением тестов.  

## 6. Ожидаемые результаты
- Консольный отчет о прохождении тестов (`TextTestRunner`):
  - Показан результат каждого теста (`ok` для успешных, описание ошибки для проваленных).
- Дефекты выявляются и документируются (если есть).
- Гарантированная корректность вычислений площади и периметра для протестированных случаев.  

## 7. Размещение исходного кода тестов
- Тесты находятся в папке `tests/rectangle_test.py`.
- Репозиторий GitHub: https://github.com/arturikq/utests
- История коммитов фиксирует появление тестов.
